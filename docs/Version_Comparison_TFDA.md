# 版本更新比較：TFDA 台灣衛福部食品營養成分資料庫整合

本專案在這個版本中進行了核心資料庫的升級，將原本手動建置的小型測試資料庫，全面升級為台灣衛福部 (TFDA) 官方公開的食品營養成分資料庫。以下為新舊版本的差異比較與重要更新項目。

## 舊版 (v0.0.1)：概念驗證階段 (PoC)

* **資料來源**：使用 `nutrition_db.json` 作為主要的資料來源，內部僅包含約 12 筆手動輸入的食品資料（例如：蘋果、披薩、熱狗等）。
* **營養數據**：僅提供最基礎的 6 項數據（熱量、蛋白質、脂肪、碳水化合物、鈉、膳食纖維）。
* **運作機制**：YOLO 影像辨識出物體後，直接以英文標籤 (Label) 對應到上述的小型 JSON 檔案。如辨識出不在資料庫內的物品，則全部歸類為「未知食物」。
* **侷限性**：資料太少、不夠在地化（例如沒有台灣特有食品）、營養數據不夠齊全，無法支援複雜的疾病過濾（例如缺乏維生素、鉀、磷等數據）。

---

## 新版 (v0.0.2)：TFDA 官方數據升級 (可以考慮再加上這個資料庫，https://consumer.fda.gov.tw/uc/GetFile.ashx?type=ServerFile&id=5721751174327419804)

* **資料來源升級**：
  * 解析了高達 128MB (226,824 筆紀錄) 的衛福部原始 JSON 資料 (`20_5.json`)。
  * 透過轉檔腳本 (`convert_tfda.py`)，整理出 **2,181 筆台灣常見食品**，並儲存為全新且輕量的 `nutrition_db_tw.json` (1.7MB)。
* **營養數據擴充**：從基礎 6 項擴增至 **28 項核心與進階營養素**（包含糖、飽和脂肪、反式脂肪、膽固醇、多種維生素、鉀、鈣、鐵、鎂、磷、鋅等），讓 AI 推薦系統與疾病禁忌過濾規則能夠精準發揮作用。
* **智慧映射層 (YOLO to TFDA Mapping)**：
  * 實作了 `yolo_tfda_mapping.py`，作為 YOLO 英文標籤與 TFDA 中文食品名稱之間的橋樑（例如將 YOLO 的 `banana` 映射到 TFDA 的 `北蕉平均值`，`hot dog` 映射到 `熱狗`）。
* **三層 Fallback 安全機制**：
  * YOLO 辨識出物品後，系統會依照以下順序查詢營養成分：
    1. 優先透過映射表尋找 **TFDA 在地資料庫**。
    2. 若無對應，退回舊版 **手動建置資料庫**。
    3. 若皆無，則回傳 **未知食物** 預設值。
* **全新 API 支援 (後端更新)**：
  * 新增 API `/search/food`：允許使用者以中文關鍵字（如：`蘋果`），在 2,181 筆 TFDA 食品中進行全文檢索，大幅強化前端「手動搜尋與添加食物」的功能。
  * 新增 API `/food/<food_key>`：可直接獲取單一食品的 28 項完整營養結構。

## 總結

本次更新在**不改變前端架構與 YOLO 模型**的前提下，將後端「營養資料引擎」徹底在地化與專業化。從原本的「玩具級資料」躍升為「醫療級官方數據」，使得本專案的「個人化疾病與飲食推薦」功能，具備了真實上線的實用價值。
