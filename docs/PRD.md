摘要:
隨著現代生活節奏加快，外食族比例逐年攀升。然而，外食成分透明度低，使用者難以即時掌握精確的營養資訊，長期攝取不均容易導致生理機能下降，並增加肥胖、糖尿病及心血管疾病等代謝症候群的風險。對於患有慢性病或特殊生理狀態的使用者而言，飲食禁忌（如高血壓者的鈉含量限制、腎臟病者的蛋白質與磷鉀控管、或糖尿病者的醣份限制）是病程管理的關鍵與難點，但傳統手動輸入效率低下，導致使用者常常無法持之以恆。 

為解決此問題，本研究開發一套結合電腦視覺與智慧決策之行動系統。系統透過影像識別即時判定食物類別，並根據使用者預設的疾病配置，自動比對潛在的飲食禁忌與過敏原。結合基礎代謝率（BMR）與每日總消耗熱量（TDEE）數據與醫療級膳食指南提供個人化推薦，達成從「監測分析」到「行動導航」的健康管理閉環，確保使用者在追求體脂管理的同時，能有效避開誘發疾病加重的負面飲食因素。

研究背景與動機:
隨著全球現代化進程加速，生活步調的極致壓縮導致「久坐不動」與不健康飲食習慣成為全球性的公共衛生挑戰 。這類生活型態與肥胖、心血管疾病、高血壓及第 2 型糖尿病等慢性疾病有著高度的正相關 。雖然現代人的健康管理意識逐漸抬頭，但在實際執行層面卻面臨巨大的資訊門檻與環境阻礙 。 

根據衛生福利部國民健康署（HPA）的統計數據，臺灣外食族比例已突破 7 成 。對於這群龐大的外食族群而言，雖然市面上充斥著各種標榜健康的餐點，但外食成分的透明度極低，一般民眾難以透過肉眼精確判斷每一餐的鈉含量、油脂比例或精緻澱粉攝取量 。這種資訊不對稱導致使用者往往在「隨意進食」帶來的短暫滿足感，與事後對健康損害的「健康焦慮」之間反覆循環 。 

在現有的解決方案中，雖有多款飲食紀錄軟體，但傳統的手動紀錄方式要求使用者必須逐項輸入食物名稱、自行查閱營養價值表，並手動估算份量 。這種極高操作門檻的紀錄過程不僅繁瑣且容易出錯，更大幅消磨了使用者的意志力，導致大多數健康管理計畫在執行初期便因「紀錄疲勞」而被迫終止 。因此，如何運用自動化技術大幅降低紀錄負擔，並將冰冷的數據轉化為具備實踐價值的行動決策，已成為當代預防醫學與資訊技術結合的關鍵課題 。

研究目的:
本研究旨在運用 AI Agent（人工智慧代理人）的概念，開發一套結合「電腦視覺辨識」與「多約束智慧決策引擎」的個人化營養導航系統。本系統不單是作為被動的紀錄工具，而是扮演具備醫療輔助意識的「數位營養師」，利用 YOLO 深度學習技術，解決傳統手動輸入效率低下且易產生誤差的痛點。透過影像識別即時判定食物類別，並結合邊界框分析實現份量估算，讓餐點紀錄達成自動化。 

系統將根據使用者的身高、體重、年齡及活動等級，動態計算基礎代謝率（BMR）與每日總消耗熱量（TDEE）。將健康管理從籠統的口號轉化為精確到大卡（kcal）與公克（g）的量化監控。 

針對患有慢性病（如糖尿病、高血壓、慢性腎臟病）之使用者，建立「硬性排除規則」。系統能自動辨識並過濾含有高升糖指數（High GI）、高鈉或特定過敏原之餐點，確保推薦內容具備醫療安全性。

AI 食物辨識與營養分析相關研究:
在食物辨識與營養分析領域中，近年來深度學習技術被廣泛應用。其中，在AI-Driven Food Tracking & Diet Recommendation System 的研究 提出一套結合深度學習與決策樹演算法的飲食管理系統。該系統利用 YOLOv8 進行即時食物影像辨識，並透過決策樹（Decision Tree）演算法分析使用者的飲食習慣，進而提供個人化飲食建議。此外，系統也能計算食物的熱量與營養攝取量，使使用者能夠即時追蹤每日飲食狀況。此研究展示了深度學習與機器學習演算法在健康飲食管理上的應用潛力。 
另一篇研究 AI Powered Food Nutrition Analyzer Using Image Recognition 則利用卷積神經網路（Convolutional Neural Network, CNN）對食物影像進行分類，並自動分析食物的營養成分，包括熱量、蛋白質、脂肪與碳水化合物等。該系統透過 Python 建立後端服務，並結合資料庫管理食物營養資訊，讓使用者只需拍攝食物照片即可取得完整營養分析結果。此架構與本研究系統所採用的技術架構相似，對於系統設計具有參考價值。
Deep Learning in Food Image Recognition: A Comprehensive Review 為一篇針對食物影像辨識技術的綜述性研究。該研究整理了近年來食物影像分類所使用的深度學習模型、常用資料集以及實際應用案例。研究指出，卷積神經網路與物件偵測模型在食物辨識上具有良好的表現，但仍面臨食物外觀相似、份量估計困難等挑戰。此研究對於了解食物辨識技術的發展與限制具有重要參考價值。

在國內研究方面，也有學生研究提出利用深度學習與行動裝置開發食物營養分析系統。該系統透過影像辨識技術判斷食物種類，並結合營養資料庫進行營養成分分析，同時提供使用者飲食紀錄與健康建議功能。此類研究顯示行動裝置與人工智慧結合已成為飲食健康管理的重要發展方向。

個人化飲食推薦與營養規劃:
除了食物辨識與營養分析外，如何提供個人化飲食建議也是相關研究的重要方向。Yum-me: Personalized Nutrient-based Meal Recommender System 提出一套基於營養需求的飲食推薦系統。該系統透過使用者的飲食偏好與營養需求建立模型，並利用推薦演算法產生適合的餐點建議。研究指出，透過個人化推薦機制，能有效提升使用者對健康飲食管理系統的使用意願。 
在AI-Based Nutrition Recommendation System 的研究則利用人工智慧技術建立個人化飲食規劃系統。系統會根據使用者的健康目標、飲食習慣與營養需求，產生一週的飲食計畫。此方法能幫助使用者更有效地控制每日營養攝取量，並維持均衡飲食。該研究在飲食推薦策略與個人化分析方面，提供了本研究系統設計的重要參考。

醫療與健康導向的 AI 飲食分析研究:
在醫療健康領域中，AI 技術也被應用於營養攝取分析。例如 AI-Based System to Assess Nutrient Intake for Patients的研究，利用影像辨識與攝取前後食物影像比較的方法，估算使用者實際攝取的營養量。該系統主要應用於醫療與營養監測領域，能協助醫療人員評估病患的飲食狀況。雖然該研究主要應用於醫療環境，但其影像分析與營養估算方法對於一般健康管理系統仍具有參考價值。 

除了學術研究外，人工智慧飲食分析系統也逐漸出現在實際應用中。例如國外研究團隊開發的 AI 食品掃描系統，可透過手機拍攝食物照片，快速分析食物熱量與營養成分。此類技術顯示 AI 與行動裝置結合在飲食健康管理領域具有高度發展潛力，也反映出市場對於智慧飲食管理工具的需求。

可行性分析:
本研究旨在開發一套基於深度學習之即時影像食物辨識與個人化飲食推薦系統，透過影像辨識技術分析使用者所攝取之食物種類，並結合營養資料庫與使用者健康資訊，提供個人化飲食建議。為確保系統開發與實際應用的可行性， 
隨著深度學習與電腦視覺技術的快速發展，影像辨識技術在食物分類領域已取得相當成熟的成果。透過卷積神經網路（Convolutional Neural Network, CNN）模型，例如ResNet、MobileNet或YOLO等架構，可有效提升食物影像辨識的準確率與效率，並能應用於即時辨識情境。 
本系統可透過建立食物影像資料集並進行模型訓練，使系統能夠辨識常見食物種類。當使用者拍攝食物影像後，系統將即時進行影像分析與分類，並結合營養資料庫計算熱量與營養成分。再進一步根據使用者的健康資訊（如年齡、性別、活動量或飲食目標），透過推薦演算法提供個人化飲食建議。

開發環境與技術:
本系統之開發環境涵蓋了前端行動應用、後端邏輯運算以及雲端資料儲存，各項技術選型皆考量到效能與跨平台的相容性。

系統開發架構:
本系統採用前後端分離架構。前端以 React Native 負責 UI 互動與影像採集；後端則透過 Flask API 整合 YOLO 物件偵測模型。資料儲存端選用 MongoDB，以應對飲食日誌中多樣化的非結構化標籤（如過敏原、疾病禁忌）。


資料庫核心設計:
為達成個人化推薦，資料庫設計以使用者生理檔案為中心，並關聯以下核心模組： 
生理與健康特徵： 儲存 BMR/TDEE 及「疾病標籤」（如糖尿病、高血壓），作為硬性過濾基準。 
視覺偏好模型： 紀錄使用者對食物影像的點擊行為，轉化為特徵向量，用於計算口味相似度。 

核心演算法實作:
即時影像辨識與量化系統接收前端上傳之影像後，由影像辨識進行多目標偵測。針對辨識出的食物邊界框（Bounding Box），系統結合預設之密度參數進行體積換算，自動填入當日攝取熱量，省去人工輸入之繁瑣。 
個人化雙軌推薦引擎 推薦邏輯分為兩個層次，確保建議既符合醫療安全又滿足個人偏好： 
安全過濾層 ： 優先讀取使用者疾病資訊，排除高風險食物（如高鈉、高 GI）。 
口味排序層： 在安全候選名單中，計算餐點與使用者偏好向量之「餘弦相似度」，並結合地圖 API 提供具備地理便利性的餐點導向。 

使用案例:
本系統之核心功能模組如下，其互動關係如下： 
即時影像辨識： 使用者透過手機相機捕捉食物影像，系統經由 影像辨識 模型即時回傳食物標籤。 
多維度營養分析： 系統根據辨識結果自動檢索標準營養基準庫，計算該份量之熱量及三大營養素。 
健康檔案與疾病管理： 使用者可設定生理參數（BMR/TDEE）及特定健康狀況（如糖尿病、高血壓、過敏原），作為系統過濾建議的基準。
個人化雙軌推薦： 結合攝取數據與健康背景，透過疾病禁忌與口味偏好提供飲食導航。 
飲食趨勢回顧： 提供圖表化的歷史紀錄，協助使用者追蹤長期的營養平衡狀態。

為實現物件導向之開發原則，本系統設計以下核心類別以確保模組之獨立性與擴充性： 
影像辨識類別 (Vision Engine)： 負責接收前端影像張量，執行物件偵測與分類，並管理辨識信心門檻。 
營養分析類別 (Nutrient Analyzer)： 處理數據轉換，將辨識標籤與估算體積映射至營養素數值。 
推薦管理類別 (Recommendation Manager)： 系統之核心決策引擎。根據剩餘營養配額與疾病禁忌規則，運算具備個人化特性的推薦清單。 
資料持久化類別 (Data Access Object)： 專責與 MongoDB 進行通訊，管理使用者檔案、偏好向量及飲食日誌。 
使用者 (User) 與 飲食紀錄 (Dietary Record)： 定義資料結構，儲存生理指標、健康標籤及帶有時間戳記的攝食數據。

系統收益:
即時化紀錄： 透過深度學習影像辨識，將原本繁瑣的手動輸入簡化為拍照紀錄，大幅降低操作門檻，提升使用者依從性。 16 
數據透明化： 即時轉換影像資訊為熱量與營養數據，讓使用者對攝取內容產生直觀感受，達成「自覺式管理」。 
長期趨勢分析： 系統彙整大數據產出週報或月報，協助使用者察覺潛在的偏好偏差或不良飲食習慣。
個人化預警機制： 結合 BMR/TDEE 與疾病約束規則，在過量攝取或錯誤攝取（如高血糖患者攝取高 GI 食物）前提供即時警示。 
精準飲食導航： 針對具備特定健康需求（如控鹽、控醣）的使用者，優先推薦符合口味且具備健康效益的替代方案。 
預防醫學應用： 從源頭控管肥胖與不當攝取誘因，長期而言能有效減少第二型糖尿病、心血管疾病等慢性病之發病風險，降低社會醫療負擔。
建立正確份量感： 透過系統的反饋機制，使用者能逐漸學習正確辨識食物種類與其對應的營養組成。 
培養自動化健康習慣： 藉由個人化推薦的持續引導，使用者能學習在不同情境（如外食、社交聚餐）下做出對健康更有利的抉擇。
